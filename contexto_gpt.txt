# Projeto de Redes de Computadores (2026-1)

## 1. Visão Geral

Este projeto de nível de pós-graduação foca na análise comparativa entre sistemas reais
(implementados via Sockets e Docker) e modelos formais de simulação (desenvolvidos
em SimPy). O objetivo é realizar uma validação cruzada para entender as discrepâncias
entre a teoria estocástica e a prática de rede.

## 2. Fase 1: Implementação Real e Inspeção

O aluno deverá implementar um sistema de transferência de arquivos utilizando a stack
Linux e os contêineres do Docker.
●​ Protocolos: Comparação entre TCP nativo e R-UDP (Reliable UDP).
●​ Controle de erros: implementação obrigatória de janela deslizante (Go-Back-N ou
Selective Repeat).
●​ Simulação de Rede: Uso do comando tc para injetar perda de pacotes e latência.
●​ Inspeção: Uso exclusivo do tcpdump para a captura de tráfego e a exportação de
logs em CSV/JSON.
●​ Cabeçalho: Campo X-Custom-Auth contendo (Matrícula + Nome).

## O projeto

O aluno deve desenvolver um par Cliente/Servidor em Python capaz de operar em dois
modos:
1.​ Modo TCP: Transferência de arquivos por meio de sockets TCP padrão.
2.​ Modo R-UDP (Reliable UDP): O aluno deve garantir a entrega íntegra do arquivo
sobre UDP, implementando:
○​ Números de sequência e confirmações (ACKs).
○​ Mecanismo de timeout e retransmissão.
○​ Checksum ou validação de integridade por bloco.

## 2.2 Ambiente de Teste e Simulação

Docker: Uso de containers Ubuntu.
●​ Controle de Tráfego (tc): É obrigatório o uso do comando tc qdisc para simular:
○​ Cenário A: 0% de perda / 10 ms de delay.
○​ Cenário B: 10% de perda / 50 ms de delay.
○​ Cenário C: 20% de perda / 100 ms de delay.

## 2.3 Validação Cruzada (A Prova Real)
O diferencial desta avaliação (fase 1) é a obrigatoriedade de confrontar os dados:
●​ Aplicação: O código em Python deve registrar o tempo de transferência e o
throughput.
●​ Rede (tcpdump): O aluno deve capturar o tráfego (.pcap) durante os testes e
exportar os dados para um arquivo CSV. No relatório, deve-se verificar se o volume
de dados e o tempo registrados pela aplicação condizem com o que foi capturado
pelo TCPDump na interface de rede.
